# Pre-Recruitment Tease — One-Liner

**Status:** Draft for review. Not for publication.
**Date:** 2026-04-30
**Use.** Single quotable line for the founder's existing social profile, posted before recruitment begins. Signals "a setup ceremony is coming" without revealing the project name or scope.
**Distinction from `2026-04-29-launch-drafts/pre-launch-tease-1.md`.** That one teases the launch event (T-2 weeks before public mint). This one teases the ceremony specifically, weeks before that.
**Length target.** ~100 characters. Mysterious but honest. Civic-monumental restraint.

A small ceremony is shortly to convene. Honest entropy welcome; the rest, in time.

## Founder notes

- **Variant A** is the warmest of the three; it reads as a quiet announcement rather than a technical one, which suits a profile post that pre-dates the recruitment emails.
- **Variant B** is the most legible to ZK-literate readers; "Phase 2" + "one honest contributor is enough" is recognisable jargon to the audience the ceremony will draw from.
- **Variant C** is the driest; reads as administrative notice. Use if the founder would prefer the post to surface no curiosity at all from non-cryptographers.
- All three avoid the project name, the protocol name, and any link. The recruitment emails go out separately and carry the full context; this post is deniable as a prelude to those.
- **Recommended cadence.** Post the tease, wait twenty-four to forty-eight hours, then begin sending recruitment emails. Anyone who has read the tease and asks first ("what ceremony?") is exactly the audience the founder wants warmed.
