# @qkb/web — Implementation Plan

> **Renamed 2026-05-03** — see [`docs/superpowers/specs/2026-05-03-zkqes-rename-design.md`](2026-05-03-zkqes-rename-design.md) for the rename baseline. Historical references to QKB/QIE/Identity-Escrow in pre-2026-05-03 commits remain immutable in git history.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Static TanStack Router SPA that generates a secp256k1 binding, produces a QES-signable JCS document, consumes a user's `.p7s`, runs `snarkjs` in a Web Worker to produce a Groth16 proof, bundles the QKB artifact, and submits `register()` to `QKBRegistry` via the user's wallet.

**Architecture:** Pure static SPA (Vite build, runnable from `file://`). All crypto/proving is client-side in Web Workers. EN + UK i18n hard-coded. The prover is swappable behind an interface so the default snarkjs implementation can be replaced with rapidsnark-wasm later.

**Tech Stack:** React 18, TanStack Router + TanStack Query, Vite, TypeScript, `@noble/secp256k1`, `@noble/hashes`, `canonicalize` (RFC 8785 JCS), `pkijs` + `asn1js`, `snarkjs`, `viem` (wallet + contract calls), `i18next` + `react-i18next`, Vitest, Playwright.

**Package dir:** `packages/web/`.

## Interface contract

Consumes:
- `@qkb/circuits-artifacts` — `.wasm`, `.zkey`, `vkey.json`.
- `public/trusted-cas/` — output of `@qkb/lotl-flattener`.
- `packages/contracts` ABI for `QKBRegistry`.

Produces:
- Downloadable `binding.qkb.json` (JCS bytes).
- Downloadable QKB artifact bundle matching orchestration §2.4.
- On-chain `register()` tx.

All errors from `lib/errors.ts` use the taxonomy from orchestration §2.5.
